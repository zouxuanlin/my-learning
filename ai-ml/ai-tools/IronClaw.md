# IronClaw - 安全优先的 OpenClaw 替代品

## 概述

IronClaw 是由 Transformer 论文作者 Illia Polosukhin（"菠萝哥"）开发的 OpenClaw 安全替代品。该项目使用 Rust 语言完全重写，旨在解决 OpenClaw 生态系统中存在的严重安全漏洞。

## 背景与动机

### OpenClaw 的安全问题
- 超过 25,000 个公开实例在没有充分安全控制的情况下暴露在互联网上
- 被安全专家称为「安全垃圾火灾（security dumpster fire）」
- 存在一键式远程代码执行、提示注入攻击、恶意技能窃取密码等漏洞
- 核心问题：当用户将邮箱 Bearer Token 交给 OpenClaw 时，会被直接送入 LLM 提供商的服务器

### 开发者背景
- Illia Polosukhin 是 2017 年「Attention Is All You Need」论文的八位共同作者之一
- 该论文提出的 Transformer 架构奠定了当今所有大语言模型的基础
- 2017 年从谷歌离职后创立 NEAR Protocol，致力于将 AI 与区块链技术融合

## 核心特性

### 安全架构
IronClaw 建立了四层纵深防御体系，最关键的特点是：**大模型本身永远接触不到原始凭证**。

- 凭证在网络边界被注入，而非传递给 LLM
- 即使大模型被提示注入攻击试图泄露凭证，凭证存储层也会直接拒绝请求
- 凭证始终加密存储，通过 CLI 访问也需要用户的系统钥匙串来解密

### 技术实现
- 使用 Rust 语言开发，利用其内存安全特性消除缓冲区溢出等传统漏洞
- 第三方技能只能在容器内部运行，无法在主机上执行脚本
- 计划进行红队测试和专业安全审查

### 部署选项
- 支持本地部署（macOS、Linux、Windows）
- 支持云端托管（机密云 confidential cloud）
- 本地部署的局限性：设备关机时智能体停止工作，移动端能耗问题
- 机密云被视为最优折中方案，提供接近本地设备的隐私保障和「永远在线」的能力

## 提示注入防护

- 当前使用启发式规则进行模式检测
- 未来目标是部署可持续更新的小型语言分类器来识别注入模式
- 开发更智能的策略系统，在不查看输入内容的情况下审查智能体的行为意图

## 生态系统愿景

IronClaw 是 NEAR Protocol 更大战略构想「用户自有 AI（User-Owned AI）」的一部分：

- 用户完全掌控自己的数据和资产
- AI 智能体在可信环境中代替用户执行任务
- NEAR 已搭建 AI 云平台和去中心化 GPU 市场等基础设施
- 开发了智能体互相雇佣的市场（market.near.ai）
